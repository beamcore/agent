# Beamcore README feature claims used in the ad

Supported product points from the current README:

- Beamcore is a terminal coding agent built on the Erlang/OTP distribution protocol.
- Every Beamcore instance is a distributed node.
- Eeva is the model-facing runtime and the only tool the model uses.
- Eeva compiles and executes Elixir AST inside a supervised worker with limits.
- Beamcore can spawn sub-agents, use inter-node RPC and hot-code reload.
- Mesh networking uses UDP broadcast and EPMD poll for discovery.
- Live runtime attach routes eval to a named target node through Erlang distribution.
- Provider system supports OpenAI-compatible APIs and runtime switching.
- Memory is backed by ETS for reads and DETS for persistence.
- Messaging gateway supports Telegram and Discord sessions.

Do not claim the README says “99% cache hit”; that is a campaign claim supplied by the owner.
